### // how I think

Most teams treat AI like a feature. I treat it like a system that needs **permissions**, **boundaries**, **feedback loops**, and **failure handling**.

The moment AI starts acting on behalf of users, you're no longer building software. You're designing behavior.

A few principles I keep coming back to:

- **Software should earn its keep.** No demos that fall apart in production. No architecture astronauting.
- **Writing is thinking.** It sharpens, filters, and purifies ideas before they become systems.
- **A problem stops being a problem when there's a solution.** Composure over panic. Solve, don't stress.
- **The dream hasn't changed since ReSharper** — tools that write code for us. We're finally close. Build accordingly.
